# Convert a Shewhart chart to an interactive plotly figure

Produces an interactive HTML plotly version of the chart that
[`ggplot2::autoplot()`](https://ggplot2.tidyverse.org/reference/autoplot.html)
would build for the same object. Useful for dashboards, reports, and any
context where hovering, zooming and panning matters.

## Usage

``` r
as_plotly(x, ...)

# Default S3 method
as_plotly(x, ...)

# S3 method for class 'shewhart_chart'
as_plotly(x, tooltip = c("x", "y"), ...)
```

## Arguments

- x:

  A
  [shewhart_chart](https://castlaboratory.github.io/shewhartr/reference/is_shewhart_chart.md)
  object.

- ...:

  Additional arguments forwarded to
  [`plotly::ggplotly()`](https://rdrr.io/pkg/plotly/man/ggplotly.html).

- tooltip:

  Character vector of aesthetics to display in the hover tooltip, as
  accepted by
  [`plotly::ggplotly()`](https://rdrr.io/pkg/plotly/man/ggplotly.html).
  Defaults to `c("x", "y")`.

## Value

A `plotly` object (S3 class `plotly` / `htmlwidget`) ready to print,
embed in a Shiny app, or save with
[`htmlwidgets::saveWidget()`](https://rdrr.io/pkg/htmlwidgets/man/saveWidget.html).

## Details

This is a separate function rather than an
[`autoplot()`](https://ggplot2.tidyverse.org/reference/autoplot.html)
argument so that loading `shewhartr` does not pull `plotly` (and its
full transitive dependency tree) into every R session that uses the
package. `plotly` lives in `Suggests`; install it explicitly if you want
to use this function.
